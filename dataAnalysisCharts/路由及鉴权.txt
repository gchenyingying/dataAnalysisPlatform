路由
1.采用前端实现，后台不需要参与
2.技术点vue-router
3.实现不同导航之间的组件切换，单页面SPA
4.后续更换加载方式为异步

鉴权
1.token，应用级别鉴权
2.前端鉴权，后台需提供token识别码
3.实现常规鉴权需求，数据流如下
a.单TAB，从未登录用户，登录页到内容页
b.单TAB，从未登录用户，内容页到登录页
c.单TAB，当前未登录，但之前登录用户，登录页到内容页,手动登录
d.单TAB，当前未登录，但之前登录用户，内容页自动登录
e.单TAB，当前未登录，但之前登录用户，自动登录失败，重新手动登录
f.多TAB，一个已登录，另一个内容页自动登录为当前账户
g.多TAB，一个已登录，另一个登录页手动登录，但前端会替换之前登录账户为后续账户，即，仅允许一个用户在同一浏览器登录的基础模式
h.单TAB，登录退出，删除当前登录账户
g.多TAB，只允许一个浏览器一个账户，一个退出登录，另一个同时退出登录

数据结构

自动登录API，因数据结构类似，后台可按需与登录API合并为一个接口
前端
{
token:String//首次登录保存的token，用于后台识别、鉴权，采用ajax方式发送，字符串化的JSON对象作为body
}

后台
{
state:Number//登录状态，0-已登录，1-用户名错误，2-密码错误，3-服务器内部原因，采用ajax方式接受，字符串化的JSON对象作为body
}
