事件分析-柱状图
根据数据管理的配置项，用户可手动选取具体维度，分类计算某一指标的累积和。

示例
用户希望统计“不同城市的订单提交量”，需如下操作：
1.进入“数据管理”，针对具体域名的app,增加订单量指标
2.进入“分析-事件分析”
2.1.选择图表样式--柱状图
2.2.配置应用数据
添加指标，点击按钮，实时从后台获取自定义指标列表，选中“订单量”；
添加维度，点击按钮，选择预定义的维度值“地区”；

*****目前版本仅支持单指标、单维度*****

2.3.选择时间范围
后台只需过滤指定时间段的数据

数据API
1.获取指标名称列表
后台发送指定应用的全部指标名称（数据管理项目中，配置的全部指标，包括页面和事件）

交互方式 ajax

前端请求结构 
空数据(目前不需要携带appId，后续需要增加字段)

后台应答结构 
Array数组（元素为指标名称，字符串类型，原始数据通过数据管理功能提交到后台），若不存在任意有效指标，返回[]

2.获取事件相关的数据
前端请求结构
{
	chartType,//图表类型 0-柱状图，后台检测到不同值，可以明确前台请求哪类数据结构
	metric,//Array，元素类型String，一个元素对应一个指标名称
	       //柱状图，只统计单一指标，length==1
	dimension,//Array，元素类型Number,0-地区 1-操作系统 2-操作系统版本 3-浏览器 4-浏览器版本
		  //柱状图，只统计单一维度，length==1
	dataFilter:[start,end]//时间范围，start开始毫秒数，end终止毫秒数
}
后台应答结构
{
	name1:value1,
	name2:value2,
	...
}
//nameX--按照请求的dimension字段，从数据库检索相关的字段值，比如,维度值3浏览器，检索数据库中的bs字段，得到"chrome" "edge"之类
//valueX--某一分类值对应的metric指标累计值，比如指标为点击量，则统计这段时间
某分类值的点击量之和
